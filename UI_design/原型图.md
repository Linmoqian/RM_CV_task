# RoboMaster 客户端布局优化方案

## 目标

- 保留图传中心视野，避免遮挡准星和敌方目标
- 关键状态一眼可读，非关键信息弱化展示
- 操作入口触达快，支持高压比赛节奏下低认知负担操作

## 全局布局（16:9）

采用“中心净空 + 四角信息岛 + 底部控制条”的 HUD 结构。

- 中心 60% 区域：仅保留准星、命中反馈、极少量瞬时提示
- 顶部中间：比赛全局状态条（局数、阶段、倒计时、比分）
- 左上：机器人生存状态（血量、脱战、热量预警）
- 右上：资源与成长（剩余弹量、缓冲能量、经验进度）
- 底部中间：性能体系切换控件（发射机构/底盘）
- 底部左侧：鼠标输入状态可视化（L/R/M、滚轮、速度）
- 右下：事件流提示（升级、过热、掉血、控制指令发送成功/失败）

## 信息优先级

P0（必须第一眼看到）：
- 当前阶段与倒计时
- 当前血量
- 过热风险
- 脱战状态

P1（次级关注）：
- 当前比分
- 弹量与缓冲能量
- 经验与升级进度

P2（按需查看）：
- 当前局号/总局数
- 历史事件日志
- 细粒度鼠标速度数值

## 组件布局建议

1. 顶部比赛条（高度 64 px）
- 左：`Round 1/3`
- 中：`阶段名 + 倒计时`（字号最大）
- 右：`Red 120 : 98 Blue`
- 暂停时显示黄色 `PAUSED` 徽标

2. 左上状态卡（宽 320 px）
- 血量条：横向主进度条 + 数值
- 热量条：接近阈值时从青色过渡到橙/红
- 脱战：开启时整卡边框红色脉冲

3. 右上资源卡（宽 320 px）
- 弹量：大号数字 + 图标
- 缓冲能量：环形或横条进度
- 经验：`xp / upgrade_xp`，满时触发升级闪动

4. 底部控制条（高度 88 px）
- 发射机构：`冷却优先` / `爆发优先` 两段式切换
- 底盘：`血量优先` / `功率优先` 两段式切换
- 当前选中状态常亮，并显示 0.2s 发送确认动画

5. 右下事件区（宽 360 px）
- 仅保留最近 3 条事件
- 新事件从下方滑入并渐隐
- 严重事件（过热、低血）使用红色 + 轻震动动画

## 视觉与可读性

- 所有卡片使用半透明深底：`rgba(8, 12, 20, 0.50)`
- 文字主色：`#EAF2FF`，次级：`#9FB3C8`
- 关键警告：`#FF5A5F`，提醒：`#FFB020`，正常：`#1FD6A1`
- 卡片边框使用 1 px 低对比线，避免在复杂图传背景中丢失轮廓
- 避免纯白大面积块，减少视觉干扰和眩光

## 动效规则

- 数值变化：120 ms 平滑补间
- 血量骤降：数字短暂放大后回落（强化危险感知）
- 过热临界：热量条每 800 ms 轻微呼吸
- 切换指令：按钮按下后出现一次“发送成功”勾形反馈

## 响应式与安全区域

- 16:9 基线设计，兼容 21:9 与 4:3 时保持中心净空不变
- 窄屏时将左右上卡片压缩为 280 px
- 最小可读字号 14 px；倒计时不低于 28 px
- 所有可交互控件与屏幕边缘至少保留 16 px 间距

## QML 实现建议（结构）

- 根节点：`Item`
- 背景层：`Image`（图传截图）
- HUD 层：`anchors.fill: parent`
- 各信息岛拆分为独立组件：
	- `TopMatchBar.qml`
	- `RobotStatusCard.qml`
	- `ResourceCard.qml`
	- `ControlBar.qml`
	- `EventFeed.qml`

通过属性绑定连接 `GameStatus` 与 `RobotDynamicStatus`，确保 UI 以数据驱动更新。
